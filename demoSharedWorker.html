SharedWorker 详解

了解如何使用SharedWorker实现浏览器多标签页之间的数据共享和通信

什么是SharedWorker？

SharedWorker是一种特殊的Web Worker，允许多个浏览器上下文（如多个标签页、iframe或窗口）共享同一个Worker实例。

与普通的DedicatedWorker（专用Worker）不同，SharedWorker可以被多个脚本共享，这使得它成为实现多标签页应用数据同步的理想选择。

核心特性

  • 共享实例：多个页面共享同一个Worker
  • 跨标签通信：实现不同标签页之间的数据同步
  • 持久连接：即使所有连接关闭，Worker仍可保持运行
  • 消息传递：通过port进行双向通信

SharedWorker通过MessagePort与每个连接它的页面进行通信。

使用场景

  • 多标签页应用状态同步
  • 共享WebSocket连接
  • 跨标签页的共享缓存
  • 后台数据同步任务
  • 共享计算密集型任务

SharedWorker特别适合需要保持单一数据源的多标签页应用。

HTML结构

创建页面并引入SharedWorker：

<!DOCTYPE html> <html> <head> <title>SharedWorker演示</title> <script> // 检查浏览器支持情况 if (!window.SharedWorker) { alert("您的浏览器不支持SharedWorker"); } </script> </head> <body> <h1>SharedWorker演示</h1> <input type="text" id="messageInput" placeholder="输入消息"> <button id="sendMessageBtn">发送</button> <div id="messageOutput"></div> <script src="main.js"></script> </body> </html>

SharedWorker代码 (shared-worker.js)

实现SharedWorker逻辑：

// 存储所有连接的端口 const ports = []; let messageCount = 0; // 处理连接 self.addEventListener('connect', function(e) { // 获取端口 const port = e.ports[0]; // 将端口添加到列表 ports.push(port); // 发送当前连接数 port.postMessage({ type: 'connection', count: ports.length }); // 处理来自页面的消息 port.addEventListener('message', function(e) { messageCount++; // 向所有连接的页面广播消息 ports.forEach(p => { p.postMessage({ type: 'message', content: e.data, count: messageCount, timestamp: new Date().toLocaleTimeString() }); }); }); // 启动端口 port.start(); // 处理端口断开连接 port.addEventListener('close', function() { const index = ports.indexOf(port); if (index !== -1) { ports.splice(index, 1); } }); });

主页面代码 (main.js)

在主页面中使用SharedWorker：

// 创建SharedWorker const worker = new SharedWorker('shared-worker.js'); // 获取DOM元素 const messageInput = document.getElementById('messageInput'); const sendMessageBtn = document.getElementById('sendMessageBtn'); const messageOutput = document.getElementById('messageOutput'); // 处理来自SharedWorker的消息 worker.port.onmessage = function(e) { const data = e.data; if (data.type === 'connection') { console.log(`当前连接数: ${data.count}`); } else if (data.type === 'message') { const messageElement = document.createElement('div'); messageElement.className = 'message'; messageElement.innerHTML = ` <strong>${data.timestamp}</strong>: ${data.content} (消息#${data.count}) `; messageOutput.appendChild(messageElement); } }; // 发送消息到SharedWorker sendMessageBtn.addEventListener('click', function() { const message = messageInput.value.trim(); if (message) { worker.port.postMessage(message); messageInput.value = ''; } }); // 启动端口通信 worker.port.start();

常见问题解答

SharedWorker与DedicatedWorker有什么区别？

DedicatedWorker与单个页面绑定，而SharedWorker可以被多个页面共享。当页面关闭时，DedicatedWorker会终止，但SharedWorker在所有连接关闭后仍可能保持运行一段时间。

如何检查浏览器是否支持SharedWorker？

if (window.SharedWorker) { // 浏览器支持SharedWorker const worker = new SharedWorker('worker.js'); } else { // 浏览器不支持SharedWorker console.log('您的浏览器不支持SharedWorker'); }

SharedWorker如何与多个页面通信？

SharedWorker通过维护一个端口列表，当收到一个页面的消息时，它可以遍历所有端口并向所有连接的页面广播消息。

如何处理SharedWorker中的错误？

可以通过监听error事件来处理SharedWorker中的错误：

worker.port.onerror = function(error) { console.error('SharedWorker错误:', error); };
